## What This Project Does

**Molty Royale AI Agent Bot** — an autonomous Python bot that plays the Molty Royale Web3 battle royale game without human intervention. It handles account creation, wallet management (two EOAs: Owner + Agent), smart contract interactions, real-time WebSocket gameplay with AI decision-making, cross-game memory, and a live web dashboard. Designed for zero-config deployment on Railway.

## Running the Bot

```bash
pip install -r requirements.txt
cp .env.example .env   # fill in AGENT_NAME, ROOM_MODE, LOG_LEVEL
python -m bot.main     # first run is interactive if TTY is available
```

Docker:
```bash
docker build -t molty-bot .
docker run --env-file .env -p 8080:8080 -it molty-bot
```

There is no test suite. No linter is configured.

## Architecture

Everything is `asyncio`-based. The bot runs a single continuous state machine loop (`bot/heartbeat.py`) that calls `GET /accounts/me`, determines the current state via `bot/state_router.py`, and routes to the appropriate handler:

| State | Action |
|---|---|
| `NO_ACCOUNT` | Create account via REST |
| `NO_IDENTITY` | Run `bot/setup/` flow (wallet → whitelist → ERC-8004) |
| `READY_FREE` / `READY_PAID` | Join a game room |
| `IN_GAME` | Run WebSocket gameplay loop |

Errors trigger exponential backoff (10s → 30s → 60s → 120s). Auth failure (401) is the only exit condition.

### Key Modules

- **`bot/heartbeat.py`** — main loop, error recovery, backoff
- **`bot/state_router.py`** — maps `/accounts/me` response to state enum
- **`bot/config.py`** — all constants, API URLs, contract addresses, env var loading
- **`bot/credentials.py`** — reads/writes `dev-agent/` files (0o600 permissions); syncs to Railway env vars via `bot/utils/railway_sync.py`
- **`bot/game/websocket_engine.py`** — connects to `wss://cdn.moltyroyale.com/ws/agent`, drives the turn loop, dispatches `agent_view` → brain → action
- **`bot/strategy/brain.py`** — priority-based action selection (heal > escape death zone > pick up rewards > attack guardians > explore > move)
- **`bot/memory/agent_memory.py`** — loads/saves `~/.molty-royale/molty-royale-context.json`; splits into `overall` (strategy, win history) and `temp` (current game state, death zones)
- **`bot/dashboard/`** — aiohttp server on port 8080; `dashboard/state.py` is a shared state object updated by both bot and server; WS push loop refreshes clients every 1.5s

### Blockchain / Web3

- **Network:** CROSS Mainnet, Chain ID 612055
- **`bot/web3/eip712_signer.py`** — signs paid room join requests
- **`bot/web3/identity_contract.py`** — registers ERC-8004 identity NFT
- **`bot/web3/whitelist_contract.py`** — checks/approves contract whitelist
- Contract ABIs live in **`bot/web3/contracts.py`**

### Credentials Persistence (Railway)

On first run the bot generates API key + two wallets and writes them to `dev-agent/`. It also pushes them to Railway Variables via GraphQL (`RAILWAY_API_TOKEN` required). On container restart, credentials are restored from env vars so wallets are never regenerated.

### Rate Limits

- REST API: 300 calls/min per IP
- WebSocket: 120 messages/min per agent; 60s server-enforced cooldown between turns
- `bot/utils/rate_limiter.py` tracks both

### Environment Variables

**User-supplied:**
- `AGENT_NAME`, `ROOM_MODE` (`free`/`auto`/`paid`), `LOG_LEVEL`, `RAILWAY_API_TOKEN`

**Auto-generated by bot on first run:**
- `API_KEY`, `AGENT_WALLET_ADDRESS`, `AGENT_PRIVATE_KEY`, `OWNER_EOA`, `OWNER_PRIVATE_KEY`

**First-run behavior flags (default `true` in non-interactive mode):**
- `ADVANCED_MODE`, `AUTO_WHITELIST`, `AUTO_SC_WALLET`, `ENABLE_MEMORY`, `AUTO_IDENTITY`
